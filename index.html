01 — Nền tảng

Online vs Offline

Trong lập trình thi đấu, truy vấn chia làm hai loại dựa trên thời điểm phải trả lời:

Tiêu chí Online Query Offline Query
Thời điểm trả lời Ngay khi nhận query Sau khi đọc tất cả query
Query sau phụ thuộc query trước CÓ THỂ KHÔNG
Tự do sắp xếp lại query KHÔNG CÓ
Ví dụ cấu trúc Segment Tree, BIT, Hash Map Mo's, CDQ, Parallel BS, DSU on Tree
Độ phức tạp điển hình O(n log n) — O(n√n) Thường tốt hơn hoặc dễ cài hơn

"Offline = tự do sắp xếp. Đây là siêu năng lực mà nhiều người bỏ quên."

— Nguyên lý cốt lõi

02 — Kỹ thuật 1

Mo's Algorithm

Mo's Algorithm là kỹ thuật xử lý offline các truy vấn đoạn [l, r] bằng cách sắp xếp query theo thứ tự đặc biệt để tổng số bước dịch chuyển con trỏ là O((n + q)√n).

O(n√n)Tổng thời gian
O(√n)Block size
O(q log q)Sắp xếp query
O(1)Add / Remove

Nguyên lý sắp xếp query

Mo's blocks — Mảng chia thành khối √n, query sắp theo block(l), rồi theo r

  • 1
    Chọn block size B = √n

    Chia mảng thành các khối kích thước B. Block của index i là i / B.

  • 2
    Sắp xếp query

    Sắp theo (block[l], r) — nếu block[l] bằng nhau: block lẻ thì r giảm, block chẵn thì r tăng (Mo tối ưu).

  • 3
    Duy trì cửa sổ [curL, curR]

    Với mỗi query, dịch chuyển curL/curR bằng cách gọi add(x) hoặc remove(x) từng bước.

  • 4
    Trả lời query theo thứ tự gốc

    Lưu đáp án vào ans[idx] rồi xuất theo thứ tự ban đầu.

mo_algorithm.cpp
// ══════════════════════════════════════════════
//  Mo's Algorithm — Đếm số phần tử phân biệt
//  O((n + q) * sqrt(n))
// ══════════════════════════════════════════════
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 1e5 + 5;
int a[MAXN], cnt[MAXN];
int ans[MAXN];
int n, q, B;
int curAns = 0;

struct Query {
    int l, r, idx;
};

void add(int x) {
    if (++cnt[a[x]] == 1) ++curAns;
}

void remove(int x) {
    if (--cnt[a[x]] == 0) --curAns;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cin >> n >> q;
    B = max(1, (int)sqrt(n));
    for (int i = 0; i < n; i++) cin >> a[i];
    vector<Query> queries(q);
    for (int i = 0; i < q; i++)
        cin >> queries[i].l >> queries[i].r,
        queries[i].l--, queries[i].r--,
        queries[i].idx = i;
    sort(queries.begin(), queries.end(),
        [](const Query& a, const Query& b) {
            int ba = a.l / B, bb = b.l / B;
            if (ba != bb) return ba < bb;
            return (ba & 1) ? a.r > b.r : a.r < b.r;
        });
    int curL = 0, curR = -1;
    for (auto& qr : queries) {
        while (curR < qr.r) add(++curR);
        while (curL > qr.l) add(--curL);
        while (curR > qr.r) remove(curR--);
        while (curL < qr.l) remove(curL++);
        ans[qr.idx] = curAns;
    }
    for (int i = 0; i < q; i++) cout << ans[i] << '\n';
}
💡 Mo's với Update (Mo 3D)

Khi có cả update + query: thêm chiều thứ 3 là thời gian (timestamp). Sắp xếp theo (block[l], block[r], t). Độ phức tạp O(n^(5/3)).

03 — Kỹ thuật 2

CDQ Divide & Conquer

CDQ (陈丹琦 — tên tác giả) là kỹ thuật chia đôi tập thao tác theo thời gian, rồi tính ảnh hưởng của nửa trái lên nửa phải bằng merge sort hoặc BIT. Thay thế các cấu trúc dữ liệu online phức tạp (Balanced BST, 2D Segment Tree).

🎯 Bài toán điển hình

3D partial order / Count inversions: Cho n điểm (a, b, c), đếm số cặp (i,j) sao cho a[i]≤a[j], b[i]≤b[j], c[i]≤c[j]. CDQ giải trong O(n log²n).

cdq_3d_partial_order.cpp
// CDQ Divide & Conquer — 3D Partial Order
#include <bits/stdc++.h>
using namespace std;
struct Point { int a, b, c, ans, id; };
int bit[100005], maxC;
void upd(int x, int v) { for (; x<=maxC; x+=x&-x) bit[x]+=v; }
int  qry(int x) { int s=0; for(; x>0; x-=x&-x) s+=bit[x]; return s; }
void cdq(vector<Point>& pts, int l, int r) {
    if (l >= r) return;
    int mid = (l+r)>>1;
    cdq(pts, l, mid); cdq(pts, mid+1, r);
    int i=l, j=mid+1;
    vector<int> toClean;
    while (i<=mid || j<=r) {
        if (j>r || (i<=mid && pts[i].b <= pts[j].b)) {
            upd(pts[i].c, 1); toClean.push_back(pts[i].c); i++;
        } else { pts[j].ans += qry(pts[j].c); j++; }
    }
    for (int x : toClean) upd(x, -1);
    inplace_merge(pts.begin()+l, pts.begin()+mid+1, pts.begin()+r+1,
        [](const Point& x, const Point& y){ return x.b < y.b; });
}

Parallel Binary Search

Khi có q query độc lập, mỗi query cần binary search trên cùng một dãy sự kiện, PBS gom tất cả binary search lại và xử lý đồng thời.

parallel_binary_search.cpp
// Parallel Binary Search - O((n+q) log^2 n)
#include <bits/stdc++.h>

const int MAXN = 1e5+5;

int  qry(int x) { int s=0; for(; x>0; x-=x&-x) s+=bit[x]; return s; }

05 — Kỹ thuật 4

DSU on Tree

DSU on Tree (Small to Large) là kỹ thuật DFS giữ lại thông tin con nặng nhất, xóa nhẹ, đạt O(n log n).

dsu_on_tree.cpp
// DSU on Tree — Đếm màu phân biệt subtree
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 1e5+5;
vector<int> adj[MAXN];
int col[MAXN], sz[MAXN], heavy[MAXN], cnt[MAXN], ans[MAXN], curAns;
void dfs_sz(int u, int p) {
    sz[u]=1; heavy[u]=-1;
    for (int v: adj[u]) if(v!=p) {
        dfs_sz(v,u); sz[u]+=sz[v];
        if(heavy[u]==-1 || sz[v] > sz[heavy[u]]) heavy[u]=v;
    }
}
void add(int u, int p, int x) {
    if(++cnt[col[u]]==1) curAns++;
    for(int v: adj[u]) if(v!=p && v!=x) add(v,u,x);
}
void clear(int u, int p) { cnt[col[u]]=0; for(int v: adj[u]) if(v!=p) clear(v,u); }
void dfs(int u, int p, bool keep) {
    for(int v: adj[u]) if(v!=p && v!=heavy[u]) dfs(v,u,false);
    if(heavy[u]!=-1) dfs(heavy[u],u,true);
    add(u,p,heavy[u]); ans[u]=curAns;
    if(!keep) { clear(u,p); curAns=0; }
}

✅ Checklist chọn kỹ thuật

Bài trên đoạn, không update, add/remove O(1)? → Mo's.
Bài 2D/3D partial order, có update và query? → CDQ.
Nhiều query cùng binary search một trục? → PBS.
Bài tổng hợp trên subtree, không update? → DSU on Tree.
